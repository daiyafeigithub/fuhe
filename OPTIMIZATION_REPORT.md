### 2.2 二维码生成功能完善

#### 优化项1：增强数据校验逻辑

**修改文件**：`backend/app/main.py`

**校验规则**（符合 guide.md 要求）：

1. **企业编码校验**：
   - 检查企业是否存在
   - 检查企业状态是否为启用

2. **院内编码校验**：
   - 检查院内编码是否存在于 HIS 同步表
   - 检查药品状态是否为启用

3. **规格校验**：
   - 格式：必须为"数字+g"（如5g、10g）
   - 数值范围：1-30（含）

4. **批号校验**：
   - 格式：纯数字
   - 长度：5-10位（含）

5. **数量校验**：
   - 类型：仅数字
   - 范围：必须大于0

6. **重量校验**：
   - 范围：0-100kg
   - 小数位：可保留1-4位

**校验代码示例**：
```python
# 规格校验
if not req.spec.endswith('g'):
    return error_response("1000", "规格格式错误，应为数字+g（如5g）")
try:
    spec_num = float(req.spec.rstrip('g'))
    if not (1 <= spec_num <= 30):
        return error_response("1000", "规格数字范围错误，应为1-30（含）")
except:
    return error_response("1000", "规格格式错误，应为数字+g（如5g）")

# 批号校验
if not (5 <= len(req.batch_no) <= 10) or not req.batch_no.isdigit():
    return error_response("1000", "批号格式错误，应为5-10位纯数字")
```

### 2.3 扫码复核接口优化

#### 优化项1：增强参数校验

**校验点**：
1. 处方号不能为空
2. 筐号不能为空
3. 二维码内容不能为空
4. 复核状态检查（已完成的处方不能继续扫码）

#### 优化项2：完善错误处理

**错误处理流程**：
1. 参数校验失败 → 返回明确的错误提示
2. 二维码解析失败 → 返回详细错误信息
3. 复核未初始化 → 返回相应提示
4. 网络离线 → 保存到本地，返回 OFFLINE_SAVED 状态

**优化代码**：
```python
# 参数校验
if not req.pres_no:
    return error_response("1000", "处方号不能为空")
if not req.basket_no:
    return error_response("1000", "筐号不能为空")

# 复核状态检查
if check_main.check_status == 2:
    return error_response("8000", "该处方已复核完成，不可再扫码")

# 二维码解析增强
try:
    qrcode_bytes = base64.b64decode(req.qrcode_content)
    try:
        raw = qrcode_bytes.decode("gb2312")
    except:
        raw = qrcode_bytes.decode("utf-8")
except Exception as e:
    return error_response("7000", f"二维码解析失败: {str(e)}")
```

**JavaScript代码**：
```javascript
function setLoading(btnId, loading) {
    const btn = document.getElementById(btnId);
    if (!btn) return;
    if (loading) {
        btn.disabled = true;
        btn.innerHTML = '<span class="loading"></span>处理中...';
    } else {
        btn.disabled = false;
        const originalText = btn.getAttribute('data-original-text') || '确定';
        btn.innerHTML = originalText;
    }
}
```
