## 一、系统登录

### 1.1 访问系统

在浏览器中访问系统地址：
- **开发环境**：http://localhost:8000/frontend/
- **生产环境**：http://[服务器IP]:[端口]/frontend/

### 1.2 登录系统

1. 输入用户账号
2. 输入用户密码
3. 点击"登录"按钮

**默认管理员账号**：
- 账号：`admin`
- 密码：`admin123`

### 1.3 首次登录

首次登录后，建议：
1. 修改管理员密码
2. 创建复核员账号
3. 配置基础数据（企业、药品等）

## 二、二维码管理

### 2.1 企业标志维护

**功能位置**：二维码管理 → 企业标志维护

**操作步骤**：
1. 输入企业编码（1-6）
2. 输入企业全称
3. 选择状态（启用/禁用）
4. 点击"保存"按钮

**注意事项**：
- 企业编码必须唯一
- 禁用的企业不可用于二维码生成

### 2.2 单条生成二维码

**功能位置**：二维码管理 → 单条生成

**操作步骤**：
1. 选择企业编码
2. 输入院内编码（CJID）
3. 输入规格（如5g、10g）
4. 输入批号（5-10位纯数字）
5. 输入数量（默认7）
6. 输入重量（kg）
7. 点击"生成二维码"按钮

**数据校验规则**：
- 规格：必须为"数字+g"格式，数字1-30（含）
- 批号：纯数字，长度5-10位（含）
- 数量：必须大于0
- 重量：0-100kg

### 2.3 批量生成二维码

**功能位置**：二维码管理 → 批量生成

**操作步骤**：
1. 下载Excel模板
2. 按模板格式填写数据
3. 上传填写好的Excel文件
4. 系统自动批量生成
5. 下载生成的二维码图片包

### 2.4 二维码解析/校验

**功能位置**：二维码管理 → 解析/校验

**操作步骤**：
1. 使用扫码枪扫描二维码（或粘贴Base64字符串）
2. 点击"解析"按钮查看二维码内容
3. 点击"校验"按钮验证二维码有效性

**校验结果**：
- **SUCCESS**：二维码与原始记录一致（有效）
- **FAIL**：二维码与原始记录不一致（无效）
- **NOT_FOUND**：未查询到二维码记录（无效）

## 三、扫码复核

### 3.1 HIS处方同步

**功能位置**：扫码复核 → 同步处方

**操作步骤**：
1. 输入处方号（单条同步）
2. 或选择时间范围（批量同步）
3. 点击"同步"按钮
4. 系统自动从HIS系统获取处方信息

**处方状态**：
- 未复核：1
- 复核中：2
- 已复核：3
- 复核异常：4

### 3.2 初始化复核

**功能位置**：扫码复核 → 初始化复核

**操作步骤**：
1. 输入处方号
2. 输入复核人员账号
3. 输入复核台编号（如T01、T02）
4. 点击"初始化"按钮

**注意事项**：
- 处方必须先从HIS同步
- 处方状态必须为"未复核"
- 初始化后处方状态变为"复核中"

### 3.3 实时扫码复核

**功能位置**：扫码复核 → 扫码

**操作步骤**：
1. 输入处方号
2. 输入筐号
3. 使用扫码枪扫描饮片二维码（或粘贴Base64字符串）
4. 系统自动显示扫码结果

**扫码结果**：
- **SUCCESS**：扫码成功，药品信息匹配
- **OFFLINE_SAVED**：网络离线，数据已保存至本地

**错误提醒**：
- 药品错误：院内编码不符/规格不符
- 数量错误：扫码数量与处方要求数量不符

### 3.4 保存进度

**功能位置**：扫码复核 → 保存进度

**操作步骤**：
1. 输入处方号
2. 输入已完成CJID列表（逗号分隔）
3. 输入未完成CJID列表（逗号分隔）
4. 输入当前筐号
5. 点击"保存"按钮

**中断续核**：
- 复核过程中可随时保存进度
- 如需中断，下次选择"复核中"的处方可继续
- 系统自动加载保存的进度

### 3.5 提交复核结果

**功能位置**：扫码复核 → 提交结果

**操作步骤**：
1. 输入处方号
2. 输入复核人员账号
3. 点击"提交"按钮
4. 系统自动校验并提交

**提交前校验**：
- 所有饮片已复核完成
- 无未处理的错误记录

**提交后结果**：
- 处方状态变为"已复核"
- 复核结果同步至HIS系统
- 不可再修改复核记录

## 四、分筐复核

### 4.1 筐号管理

**功能位置**：分筐管理 → 维护筐号

**操作步骤**：
1. 输入筐号（系统自动生成或人工自定义）
2. 输入筐名/备注（可选）
3. 选择状态（启用/作废）
4. 点击"保存"按钮

**筐号生成规则**：
- **系统生成**：K + 8位日期 + 3位序号（如K20260228001）
- **人工自定义**：数字/字母/组合，长度≤10位

### 4.2 饮片分筐关联

**功能位置**：分筐管理 → 筐-饮片关联

**操作步骤**：
1. 输入处方号
2. 输入筐号
3. 输入CJID列表（逗号分隔）
4. 点击"关联"按钮

### 4.3 分筐复核确认

**功能位置**：分筐管理 → 确认筐

**操作步骤**：
1. 输入处方号
2. 输入筐号
3. 输入复核人员账号
4. 点击"确认"按钮

**注意事项**：
- 单筐确认后，该筐状态变为"已复核"
- 所有筐确认后，处方才可提交

## 五、错误提醒

### 5.1 错误记录查询

**功能位置**：错误提醒 → 查询

**操作步骤**：
1. 输入处方号（可选）
2. 选择错误状态（可选：待处理/已处理）
3. 点击"查询"按钮
4. 查看错误记录列表

**错误类型**：
- **DRUG_ERROR**：药品错误（院内编码不符/规格不符）
- **NUM_ERROR**：数量错误（扫码数量≠处方要求数量）

### 5.2 错误处理

**错误提醒方式**：
1. 终端弹窗（红色高亮）
2. 声音警报（可关闭）
3. 终端震动（可关闭）
4. 界面高亮（红色标注错误条目）

**处理结果类型**：
- **REPLACE**：错抓更换
- **ADD**：少包补加
- **CANCEL**：误扫取消

**操作步骤**：
1. 查看错误提醒详情
2. 根据实际情况处理错误
3. 记录处理结果
4. 错误记录状态变为"已处理"

## 九、离线模式

### 9.1 离线模式说明

系统支持完整的离线操作，当网络中断时：

1. **自动切换**：系统自动检测网络状态并切换到离线模式
2. **数据缓存**：所有操作数据保存到本地数据库
3. **无缝操作**：离线时可以正常进行扫码复核、进度保存等操作

### 9.2 离线操作功能

**支持离线的操作**：
- ✅ 扫码复核
- ✅ 进度保存
- ✅ 复核提交
- ✅ 二维码解析

### 9.3 数据同步

**自动同步**：
- 网络恢复后，系统自动触发离线数据同步
- 同步顺序：扫码记录 → 进度记录 → 提交记录
- 同步失败会自动重试（最多10次）

**手动同步**：
- 调用离线数据同步接口
- 查看同步结果（成功数/失败数）

## 十、常见问题

### Q1: 忘记密码怎么办？

**A**: 联系系统管理员重置密码。

### Q2: 二维码生成失败怎么办？

**A**: 检查以下内容：
1. 企业编码是否已维护且启用
2. 院内编码是否在HIS系统中存在
3. 规格、批号等参数格式是否正确

### Q3: 扫码后提示错误怎么办？

**A**: 根据错误类型处理：
1. **药品错误**：检查饮片是否正确，错抓则更换
2. **数量错误**：检查数量是否正确，少包则补加

### Q4: 复核中断了怎么办？

**A**: 
1. 选择"复核中"的处方
2. 系统自动加载历史保存的进度
3. 继续进行扫码复核

### Q5: 网络中断了怎么办？

**A**: 
1. 系统自动切换到离线模式
2. 正常进行扫码复核等操作
3. 网络恢复后数据自动同步

### Q6: 如何查看自己的复核记录？

**A**: 
1. 进入溯源管理模块
2. 输入复核人员账号
3. 点击"查询"按钮
4. 查看操作记录列表

### Q7: 如何生成工作量报表？

**A**: 
1. 进入工作量统计模块
2. 选择统计维度和查询条件
3. 点击"查询"按钮查看统计结果
4. 点击"生成报告"按钮下载报表

## 十二、操作建议

### 12.1 扫码复核建议

1. **准备工作**：
   - 确保处方已从HIS同步
   - 初始化复核并绑定复核台
   - 准备好扫码设备

2. **扫码顺序**：
   - 按处方清单顺序扫码
   - 每扫一个立即查看结果
   - 注意错误提醒提示

3. **分筐管理**：
   - 根据煎药要求合理分筐
   - 单筐复核完成及时确认
   - 所有筐确认后再提交

4. **异常处理**：
   - 遇到错误立即处理
   - 不确定时暂停复核
   - 及时保存进度

### 12.2 数据管理建议

1. **定期备份**：建议每日备份系统数据
2. **日志清理**：定期清理过期的操作日志
3. **账号管理**：及时禁用离职人员账号

### 12.3 系统维护建议

1. **定期检查**：定期检查设备状态
2. **网络监控**：监控网络连接状态
3. **数据同步**：确保离线数据及时同步
